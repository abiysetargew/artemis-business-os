# Deploying Artemis Business OS to Production

This is a complete, no-experience-required walkthrough for getting Artemis online for free using **Render** (backend) + **Vercel** (Flutter web). Total time: ~30 minutes.

## What you'll deploy

| Component | Host | Cost |
|---|---|---|
| NestJS backend (PostgreSQL via Neon) | Render Web Service | Free |
| Flutter web app | Vercel | Free |

## Prerequisites

- A GitHub account (you have one — your code is already there)
- A Render account (free — https://render.com, sign up with GitHub)
- A Vercel account (free — https://vercel.com, sign up with GitHub)
- A Neon account (free — https://neon.tech, sign up with GitHub)

---

## Part 1 — Set up the production database (Neon)

1. Go to https://console.neon.tech
2. Create a new project: name `artemis-prod`, region closest to your users
3. Once created, go to **Connection Details** → copy the **Pooled connection string**. It looks like:
   ```
   postgresql://artemis_owner:AbCdEf123456@ep-cool-darkness-123456.us-east-2.aws.neon.tech/artemis?sslmode=require
   ```
4. **Save this URL** — you'll paste it into Render in the next step.

---

## Part 2 — Deploy the backend to Render

1. Go to https://dashboard.render.com → **New +** → **Blueprint**
2. Connect your GitHub repo (`Artemis`)
3. Render will detect `render.yaml` and show the `artemis-backend` service
4. Click **Apply**
5. In the Environment Variables section, **manually set**:
   - `DATABASE_URL` = the Neon connection string from Part 1
   - The `JWT_SECRET` and `JWT_REFRESH_SECRET` will be auto-generated by Render (`generateValue: true`)
6. Click **Create Web Service**
7. Wait 3-5 minutes for the first build. You can watch the logs live.
8. Once deployed, copy your backend URL. It will be something like:
   ```
   https://artemis-backend-xxxx.onrender.com
   ```
9. **Test it**: open `https://artemis-backend-xxxx.onrender.com/api/v1/health` in a browser — you should see `{"status":"ok",...}`
10. **Seed the production database** (one-time):
    - Go to your service in Render → **Shell** tab (top right)
    - Run: `npx ts-node prisma/seed.ts`
    - Wait ~30 seconds. You should see "Database seeded successfully!"
    - This creates 8 BOMs, 5 customers, 30+ products, 2 users (`admin@artemis.com / admin123`, `sales@artemis.com / user123`)

> ⚠️ **Note on Render free tier**: services spin down after 15 minutes of inactivity. The first request after spin-down takes ~30-60 seconds. Subsequent requests are fast. For 24/7 availability, upgrade to the $7/month plan.

---

## Part 3 — Deploy the Flutter web app to Vercel

1. **First, edit `vercel.json` at the repo root** to point to your real backend URL:
   ```json
   "buildCommand": "cd mobile && flutter build web --release --dart-define=API_BASE_URL=https://artemis-backend-xxxx.onrender.com/api/v1",
   ```
   Replace `artemis-backend-xxxx` with your actual Render URL.

2. Commit and push:
   ```bash
   git add vercel.json
   git commit -m "Configure production API URL"
   git push
   ```

3. Go to https://vercel.com/dashboard → **Add New** → **Project**
4. Import your `Artemis` GitHub repo
5. Vercel will auto-detect `vercel.json`. Verify the settings:
   - **Build Command**: `cd mobile && flutter build web --release --dart-define=API_BASE_URL=https://artemis-backend-xxxx.onrender.com/api/v1`
   - **Output Directory**: `mobile/build/web`
   - **Install Command**: `cd mobile && flutter pub get`
6. Click **Deploy**
7. Wait 5-8 minutes (Flutter web builds are slow the first time)
8. Once deployed, you'll get a URL like:
   ```
   https://artemis-xxxx.vercel.app
   ```
9. **Test it**: open the URL → you should see the login screen → log in with `admin@artemis.com / admin123`

---

## Part 4 — Final verification

After both services are up:

1. Open the Vercel URL
2. Log in with `admin@artemis.com` / `admin123`
3. Create a customer
4. Create a sale
5. Record a payment
6. Record a production batch (uses a BOM)
7. Open Reports → see real KPIs

If anything fails, the most common causes:
- **CORS error in browser console** — backend already allows any origin, so unlikely
- **"Failed to load dashboard data"** — backend is sleeping (Render free tier). Wait 30s and refresh.
- **"Network error"** — wrong `API_BASE_URL` in `vercel.json`. Re-deploy with the right URL.

## Ongoing operations

### Adding users
- Mobile: Dashboard → **Users** (admin only) → **+** button
- Or: Render shell → `npx ts-node scripts/create-user.ts <email> <password> <role>`

### Updating the backend
```bash
git add -A
git commit -m "..."
git push
```
Render auto-deploys from `main`.

### Updating the web app
```bash
git add -A
git commit -m "..."
git push
```
Vercel auto-deploys from `main`.

### Checking logs
- **Backend logs**: Render dashboard → your service → **Logs** tab
- **Web app logs**: Vercel dashboard → your project → **Logs** tab

### Rotating secrets
1. Render dashboard → your service → **Environment** → edit `JWT_SECRET` and `JWT_REFRESH_SECRET`
2. Click **Save** → Render restarts the service
3. All users are logged out (they have to re-login)

### Resetting the database
1. Neon console → your project → **Reset** (this drops all data)
2. Render shell → `npx ts-node prisma/seed.ts`

## Free-tier limitations & workarounds

| Limitation | Workaround |
|---|---|
| Backend spins down after 15 min idle | Upgrade to $7/mo, or accept ~30s delay on first request |
| Web has no build cache on first deploy | Subsequent deploys are 1-2 min |
| Free Postgres has 0.5 GB storage | Plenty for an ERP with thousands of records |
| No custom domain (free) | Buy a domain on Namecheap (~$10/yr) and point it |
